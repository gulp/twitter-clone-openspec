+### 1.13 Media Upload Ownership and Finalization
+Direct client uploads must not allow arbitrary public URLs to be attached to tweets or profiles.
+
+**Decision:** introduce a `MediaUpload` table. The flow becomes:
+1. client requests an upload intent
+2. server creates a `MediaUpload` row and returns a presigned URL scoped to that object key
+3. client uploads the file
+4. client calls `media.confirmUpload`
+5. server performs a `HEAD` check (content type, size, existence, owner prefix) and marks the row `UPLOADED`
+6. tweet/profile mutations accept upload IDs, not raw URLs
+
+This removes trust in client-supplied `mediaUrls`, makes ownership enforceable, and gives us a clean place to garbage-collect orphaned objects.

+### 1.14 Deletion Semantics
+Soft delete needs to be explicit to avoid privacy and counter bugs.
+
+**Decision:** tweet deletion sets `deleted = true`, `deletedAt = now()`, clears rendered content/media references, and decrements any denormalized counters affected by that tweet in the same transaction:
+- author `tweetCount`
+- parent tweet `replyCount` if the deleted tweet was a reply
+
+Deleted tweets remain addressable by ID so reply threads and quote-tweet embeds can render a tombstone rather than a broken reference.
+
