# ID Generation Strategy

## What

The project uses CUID for all database primary keys and CUID2 for S3 object keys. CUID provides URL-safe, roughly time-ordered identifiers that are more readable than UUIDs. Tweet URLs appear as `/status/clx9abc123` rather than `/status/550e8400-e29b-...`.

## Where

**Prisma schema (CUID):**
- `prisma/schema.prisma:12` — User.id `@default(cuid())`
- `prisma/schema.prisma:42` — Tweet.id `@default(cuid())`
- `prisma/schema.prisma:110` — Like.id `@default(cuid())`
- `prisma/schema.prisma:137` — Retweet.id `@default(cuid())`
- `prisma/schema.prisma:152` — Notification.id `@default(cuid())`

**OAuth username generation (CUID):**
- `src/server/auth.ts:197-198` — imports `cuid` package, generates userId with `cuid()`
- `src/server/auth.ts:208` — username suffix uses first 6 chars of CUID

**S3 object keys (CUID2):**
- `src/server/trpc/routers/media.ts:3` — imports `createId` from `@paralleldrive/cuid2`
- `src/server/trpc/routers/media.ts:49-50` — generates S3 key as `{purpose}/{userId}/{createId()}.{ext}`

## How It Works

**Database primary keys** are generated by Prisma's `@default(cuid())` function, which uses the `cuid` package internally. When a model is created without an explicit `id`, Prisma generates a CUID automatically:

```typescript
// Prisma auto-generates id via @default(cuid())
const tweet = await prisma.tweet.create({
  data: { content: "Hello", authorId: userId }
});
// tweet.id is a CUID like "clx9abc123def456"
```

**OAuth username generation** explicitly imports the `cuid` package to generate a userId that matches Prisma's format, then uses the first 6 characters as a username suffix:

```typescript
// src/server/auth.ts:197-208
const cuid = (await import("cuid")).default;
const userId = cuid();  // e.g., "clx9abc123def456"
const username = `${baseUsername}_${userId.slice(0, 6)}`;  // e.g., "johnsmith_clx9ab"
```

This ensures the username suffix is derived from the same CUID that will become the User.id when the record is created.

**S3 object keys** use CUID2 from `@paralleldrive/cuid2` to generate unique filenames for uploaded media:

```typescript
// src/server/trpc/routers/media.ts:49-50
const fileId = createId();  // CUID2, e.g., "tz4a98xxat96iws9zmbrgj3a"
const key = `${purpose}/${userId}/${fileId}.${ext}`;  // e.g., "tweet/clx9abc123/tz4a98xxat96.jpg"
```

CUID2 is used here because:
1. S3 keys are not database primary keys
2. CUID2 has stronger collision resistance (128-bit vs 96-bit entropy)
3. S3 key generation is higher volume than database record creation

## Invariants

**I1.** All database primary keys MUST use Prisma's `@default(cuid())` (CUID format).

**I2.** OAuth username generation MUST use the `cuid` package to match Prisma's ID format.

**I3.** S3 object keys MAY use CUID2 (`@paralleldrive/cuid2`) because they are not database identifiers.

**I4.** IDs generated by Prisma are 25 characters long (CUID format).

**I5.** IDs generated by CUID2 are 24 characters long by default.

**I6.** Both CUID and CUID2 IDs are URL-safe (no special encoding required).

## Gotchas

**G1. Do not import `@paralleldrive/cuid2` for database primary keys.** Prisma's `@default(cuid())` uses the `cuid` package. If you manually generate an ID before creation, use `import cuid from "cuid"`, not `import { createId } from "@paralleldrive/cuid2"`.

**G2. CUID and CUID2 have different formats.** CUID starts with `c` and is 25 characters. CUID2 does not have a fixed prefix and is 24 characters by default. Do not assume IDs follow a specific pattern for validation.

**G3. The OAuth username suffix is 6 characters, not the full CUID.** Code at `src/server/auth.ts:208` uses `userId.slice(0, 6)` to keep usernames within the 15-character limit. The full userId is stored in the database, but only a prefix appears in the username.

**G4. S3 keys mix CUID and CUID2 in the same path.** A typical S3 key looks like `tweet/clx9abc123def456/tz4a98xxat96iws9.jpg` where the userId is CUID (from database) but the fileId is CUID2. This is intentional and correct.

**G5. Do not use CUID for cryptographic purposes.** Neither CUID nor CUID2 are cryptographically secure random number generators. Use `crypto.randomBytes()` for tokens, session IDs, or security-sensitive identifiers (e.g., password reset tokens at `src/server/trpc/routers/auth.ts:201`).
